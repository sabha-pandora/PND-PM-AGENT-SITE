What the PM Agent Does

An AI-powered product management assistant that integrates directly with any Pandora team's Jira board and Confluence workspace via REST API. Switch teams by changing one env variable.

✍

Write User Stories

Jira REST API

Gherkin BDD

Auto-Response

Confluence API

Quality Framework

Built-in Best Practices

Every action the agent takes is guided by Pandora Digital's quality engineering standards.

☑ INVEST Criteria

  • Independent — self-contained, no hidden dependencies
  • Negotiable — describes what, not how
  • Valuable — delivers clear user/business value
  • Estimable — enough detail to size
  • Small — fits in one sprint (1-3 days)
  • Testable — clear pass/fail acceptance criteria

☑ Gherkin BDD Format

  • Given a precondition or context
  • When an action is performed
  • Then the expected outcome occurs
  • Minimum 3 scenarios per story
  • Happy path + edge case + negative path
  • Atomic: one behavior per scenario

☑ SPIDR Decomposition

  • Spike — research unknowns first
  • Paths — split by user workflows
  • Interfaces — split by channels
  • Data — split by variations
  • Rules — split by business rules

☑ Anti-Pattern Detection

  • Rejects vague language ("works correctly")
  • Catches solutions-in-disguise
  • Flags missing negative/error paths
  • Splits compound criteria automatically
  • Identifies assumed context

☑ Definition of Ready

  • As a / I want / So that format
  • ACs reviewed by QE
  • Sized and estimated by dev team
  • Dependencies documented
  • Mockups & API contracts attached
  • Linked to parent Epic

☑ Definition of Done

  • All ACs met and signed off by PO
  • Unit tests passing (min 80% coverage)
  • Functional tests executed & linked
  • No open Critical/High defects
  • Code peer reviewed & merged
  • Docs and release notes updated
Impact

Benefits

How the PM Agent transforms product management workflow for the PVA team.

1

Consistent Story Quality

Every story follows the same format, passes INVEST criteria, and includes Gherkin BDD acceptance criteria — eliminating quality variance across PMs.

2

Faster QE Handoff

Given/When/Then ACs translate directly into test cases. QE can write automated tests from day one without ambiguity or back-and-forth.

3

Reduced Grooming Time

Stories arrive at grooming pre-checked against Definition of Ready. Teams spend time discussing value, not fixing formatting.

4

Instant Query Resolution

Dev/UX/QE questions get contextual answers within minutes, not days. The agent cross-references ACs, related stories, and Figma designs automatically.

5

Zero Scope Creep

Clear ACs define exactly what's in and out of scope. Anti-pattern detection catches gold-plating and compound criteria before they enter a sprint.

6

Living Documentation

Confluence docs are generated directly from Jira data — always current, always structured, never manually copy-pasted.

7

Design-to-Story Pipeline

Figma designs are automatically translated into stories and ACs, ensuring nothing gets lost between design handoff and implementation.

8

Board-Wide Visibility

Quality audits and knowledge base sync give real-time visibility into AC coverage, test gaps, and story quality across all 40 epics.
